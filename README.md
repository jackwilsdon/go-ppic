Profile picture generation service written in Go. A demo can be found at [ppic.now.sh](https://ppic.now.sh/hello).

`go-ppic` provides two commands; [`ppic`](#ppic) and [`ppicd`](#ppicd).

## ppic

`ppic` is used to generate profile pictures on the command line, without having to run a web server. `ppic` outputs the generated image to stdout.

### Installation

```Shell
go get -u github.com/jackwilsdon/go-ppic/cmd/ppic
```

### Usage

```Shell
ppic username [size] > image.png
```

> `size` defaults to 512 if not provided

### Examples

```Shell
ppic jackwilsdon 1024 > profile.png
```

## ppicd

`ppicd` is a web server providing image generation.

### Installation

```Shell
go get -u github.com/jackwilsdon/go-ppic/cmd/ppicd
```

### Usage

```Shell
ppicd
```

There are a number of flags that can be used to change how `ppicd` operates.

| Name      | Description                                                                                  | Default     |
|---------- | -------------------------------------------------------------------------------------------- | ----------- |
| `d`       | Whether or not the [pprof](https://golang.org/pkg/net/http/pprof/) routes should be enabled. | `false`     |
| `g`       | Whether or not responses should be gzipped.                                                  | `false`     |
| `p=xxxx`  | The port to run the server on.                                                               | `3000`      |
| `h=xxxx`  | The hostname for the server to run on.                                                       | `localhost` |

Once the server is running, you can retrieve an image by visiting `http://localhost:3000/` (provided you haven't changed `PORT` or `HOST`). Other supported file extensions are `.gif` and `.jpg` (by using `http://localhost:3000/image.gif` or `http://localhost:3000/image.jpg`), defaulting to `.png` if there is no file extension in the URL.

You can change the size of images which are generated by adding `?size=1024` to the end of the URL.
